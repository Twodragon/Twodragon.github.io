Sergey Efimochkin's Distributed File System

Description

This is a distributed file system with three components: Master Server, Slave Servers and Clients. Master Server is a mediator between Slaves and Clients, which means that it processes requests by clients, gets file data from slave servers if needed, and than sends an answer back to the client. No file data is stored on the Master Server, only files and directories names; all the files are stored on Slave Servers. Master Server and one Slave Server are already running, so you can connect, and you can run a Slave Server instance to see that files are stored there too (Though Slave Server to store data is chosen randomly, so it can happen not on the first try).

Dependecies

This project uses boost, so you have to install Boost (at least version 1.65.1), even for the client application

Download

You can download the application's code here: DFS Applications

Install and Run

To install and run the application just cd into the directory of application you want to use and run: $ cmake CMakeLists.txt
$ make
$ ./DFS_Application (DFS_client for client application, for example)

How to Interact with the DFS

The Distributed File System supports the following commands:
ls to list files in current directory
cd $name$ to go to directory named $name$
mkdir $name$ to create directory named $name$
touch $name$ $file text$ to create a file named $name$ with text of file $file text$
rmdir $name$ to delete directory named $name$ and all files in it
rm $name$ to delete file named $name$ from directory
read $name$ to print text of file named $name$
import $outer name$ $inner name$ to import file named $outer name$ from computer's file system and save it as file named $inner name$ in distributed file system
export $inner name$ $outer name$ to export file named $inner name$ into computer's file system as a file named $outername$
help for this commands list
exit to exit
Note that you cannot delete a file if somebody else is reading (downloading it) and you cannot delete a directory if somebody else is inside it.
